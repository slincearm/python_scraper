# Goodinfo 多項目自動區間爬蟲工具 (Automating Goodinfo Stock Scraper)

這是一個基於 Python 與 Selenium 構建的自動化網頁爬蟲應用程式，專門用於從 Goodinfo 網站持續、穩定地獲取特定股票的歷史區間資料。

## 🌟 功能特色
* **圖形化使用者介面 (GUI):** 使用 `tkinter` 提供簡單易用的圖形介面，支援同時輸入**多筆股號**（以分號區隔），並可輕鬆選擇年份上限及爬取項目。
* **多維度資料抓取:** 支援自動獲取「個股K線」、「本淨比」、「本益比」、「法人買賣」及「乖離率」等長期的財務數據。
* **智慧下拉選單支援:** 如果擷取項目的網頁內含有下拉選單（例如法人買賣的多種身分），系統會自動掃描並輪詢所有可用選項分別切換下載。
* **自動環境建置:** 程式啟動時會自我診斷，自動安裝缺失的第三方依賴套件 (如 `selenium`, `webdriver-manager`, `beautifulsoup4`)。
* **無縫接軌連續爬取與瀏覽器共用:** 結合 CDP 攔截檔案路徑，批次跑多檔股票時共用同一個瀏覽器實體以節省資源，途中單一股票出錯也能安全跳過。
* **智慧增量抓取與自動去重合併:** 自動偵測歷史紀錄日期，智慧鎖定最新區間進行增量爬取，並將同選項的暫存切片精準分類、合併回歷史紀錄檔後自動清理暫存。
* **資料清洗與格式正規化:** 內建強大的二維陣列表格演算法，將混亂的 HTML/XLS 轉換成結構純淨標準的 CSV 檔，完美修復原表因 colspan/rowspan 所造成的錯位現象。
* **非同步背景執行:** 爬蟲邏輯與 I/O 寫入於背景執行緒中運行，確保 GUI 持續保持響應。
* **防封鎖機制 (最新):** 特製爬蟲節奏間隔與偽裝選項，降低遭網站封鎖的風險。使用者現在可直接透過介面彈性設定每次爬取間的最高等待秒數。
* **TWSE 財報智慧雙重驗證與優化:** 在向公開資訊觀測站發起請求前，會預先掃描比對本機已存在之季度檔案，精準計算出缺乏的季度後才發送最小化 HTTP 請求，徹底省下非必要連線與重複下載；同時修復正常網頁內含隱藏標籤所誘發的「防封鎖誤判」瑕疵，並擴充文件名稱寬容度以完美支援包含「母子公司合併報表」之 101 年度前早期歷史檔案。

## 🚀 快速開始

### 前置要求
* 系統中需具備 **Python 3.x** 環境。
* 使用者電腦需安裝有 **Google Chrome** 瀏覽器。

### 執行方式
無需手動預先安裝套件，直接透過 Python 啟動主程式即可（程式內會自動偵測並 `pip install` 基本需求）：
```bash
python scraper_app.py
```

### 介面操作步驟
1. **輸入股號:** 在「股號」欄位輸入台股股票代碼。支援多檔股號以分號 `;` 隔開 (例如：`2330; 2317; 1210`)。
2. **抓取年數上限:** 設定需要往前追溯的年份數量 (預設為 `5` 年)。
3. **最大冷卻時間(秒):** 為防止伺服器封鎖，可設定最高等待秒數 (預設為 `90` 秒，系統防呆強迫最短為 60 秒)。每次抓取之間會隨機抽取 20 秒至您設定的上限進行閒置。
4. **選擇擷取項目:** 勾選需要抓取的資料項目。
5. 點擊綠色的 **「開始執行自動推迴圈爬蟲」** 按鈕。
6. 在下方日誌 (Log) 區塊觀察進度，完成後可至資料夾檢視成果。

## 📂 檔案輸出存放
全自動抓取完成的 CSV 檔案會自動創建於同層目錄的 `Download_Data` 資料夾內，並以「股號」與「擷取項目」作動態分層儲存：
```text
Download_Data/
├── 2330/
│   ├── 個股K線/
│   │   └── 個股K線_2026-04-13_Result.csv
│   └── 法人買賣/
│       ├── 外國機構投資者_2026-04-13_Result.csv
│       ├── 本國證券投資信託基金_2026-04-13_Result.csv
│       └── ...
└── 1210/
    └── 本淨比/ ...
```
暫存的切片檔案會以 `{選項名稱}_{起始日期}_{結束日期}.csv` 儲存，合併後的最終檔案則自動轉為 `{選項名稱}_{結束日期}_Result.csv` 並清除處理完畢的切片。

## 📦 打包為獨立執行檔 (.exe)
若要將此爬蟲工具打包成不需安裝 Python 環境即可點擊執行的獨立檔案，請使用 `PyInstaller`。本腳本已內建嚴密的目錄捕捉機制與 stdout 防呆，請放心隱藏終端機黑視窗。

**編譯指令 (請於專案根目錄下執行)：**
```bash
pyinstaller --onefile --noconsole --clean --exclude-module jedi --hidden-import selenium.webdriver.chrome.webdriver --hidden-import webdriver_manager scraper_app.py
```

* **參數說明：**
  * `--onefile`: 將所有依賴模組全部塞進一個乾淨的 `.exe` 檔中。
  * `--noconsole`: 隱藏背景黑色命令提示字元視窗。腳本內部已攔截 `sys.stdout` 避免了無 Console 導致的當機錯誤。
  * `--exclude-module jedi`: 強制排除非必要的程式碼補全套件 `jedi`，避免因路徑過長引發 `Failed to extract` 的解壓縮報錯問題，也能縮減檔案大小。
  * `--hidden-import ...`: 明確宣告動態隱藏引入的 webdriver 模組，避免執行時拋出 `ModuleNotFoundError` 閃退。
